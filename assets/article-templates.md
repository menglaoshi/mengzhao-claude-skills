# 文章结构模板库

读这个文件的场景：写一篇完整文章前 / 卡在结构上 / 改稿子需要重构。

---

## 总原则

**结构 = 节奏 = 完读率**

公众号读者在手机上滑动，平均每 300-500 字会产生一次"要不要继续"的判断。结构的核心任务就是在每个判断点都给读者一个"继续读"的理由。

下面 5 个结构是 2025-2026 公众号验证度最高的模板，按使用频率排序。

---

## 结构 A：故事引子型（推荐：所有 AIGC 案例 + 个人经历类）

**适用**：AIGC 落地案例、个人转型故事、真实人物深度

**总字数**：2000-3500 字

**结构骨架**：

```
[1] 故事开场（200-300 字）
  → 一个具体的、有画面感的场景
  → 不解释背景，直接扔读者进画面
  → 结尾留一个钩子或悬念

[2] 引出问题 / 反差（80-150 字）
  → 把故事引到"但是这背后其实..."
  → 抛出全文的核心问题或核心洞察

[3] 解答主体（1500-2500 字，分 3-5 段）
  → 每段一个小标题
  → 每段 300-500 字
  → 每段都要有：观点 + 论据 + 例子
  → 段与段之间用"那么"、"更重要的是"、"但还有一个细节"做过渡

[4] 可复用 SOP / 框架（200-400 字）
  → 把上面的分析浓缩成 3-5 步可操作的清单
  → 让读者觉得"看完就能用"

[5] 升华金句结尾（80-150 字）
  → 回到开头的故事
  → 用一句金句定调，让读者愿意截图转发
```

**示范开头**：

> 上周五晚上，我打开微信发现一个陌生号发来 3 万块的转账。
>
> 这是我朋友老 K 还给我的"借款"——三个月前他被裁员，找我借了 1 万块过渡。
>
> 他用这 1 万块没买股票、没投资，就买了几个 AI 工具的会员，外加一台二手 MacBook。
> 
> 三个月后，他还回来 3 万。
>
> 当他演示给我看他到底用 AI 干了什么的时候，我才意识到——大部分人对"AI 副业"的理解，都错得离谱。

## 结构 B：观点拆解型（推荐：所有方法论 + 行业观察 + 审美深度）

**适用**：理论方法论、行业趋势、审美哲学、跨界观察

**总字数**：2000-3000 字

**结构骨架**：

```
[1] 反常识开场（150-250 字）
  → 抛出一个反主流的判断
  → 用一句话激起读者"是吗？"的反应
  → 立刻给出"我为什么这么说"的铺垫

[2] 核心观点（50-100 字）
  → 一段话明确文章的核心论点
  → 这段话本身就要是金句级别（值得转发）

[3] 三段论证（每段 400-600 字 + 小标题）
  → 论据 1：[小标题]
    - 200 字阐述
    - 200 字举具体例子或案例
    - 一句话总结回到核心观点
  → 论据 2：（同上结构）
  → 论据 3：（同上结构）

[4] 反方意见 + 你的回应（200-300 字）
  → "有人会说 XXX，但我的看法是..."
  → 这一步建立信任 + 显示思考深度

[5] 总结金句 + 行动召唤（100-200 字）
  → 把核心观点用更精炼的方式重述一遍
  → 给一个让读者"代我立言"的句子作为结尾
```

**示范开头**：

> 所有人都在说"少即是多"。
> 
> 但我观察这三年的设计趋势后发现一件让我不太舒服的事：
> 
> **"极简"已经卷成了一种新的庸俗。**
>
> 真正高级的审美，从来不是"少"，而是"克制中的丰富"。

## 结构 C：清单 / 指南型（推荐：工具测评 + 学习路径 + 资源合集）

**适用**：N 个工具测评、X 步学习路径、Y 个 prompt 合集、Z 个艺术家档案

**总字数**：2500-4000 字

**结构骨架**：

```
[1] 价值前置（150-200 字）
  → 直接告诉读者：你看完这篇能得到什么
  → 不要在开头铺垫，直接抛"硬货"

[2] 主体（每条 200-400 字）
  → 用数字编号（01、02、03 而不是 1、2、3，会更显得有质感）
  → 每条结构：
    a. 数字 + 关键词标题
    b. 核心说明（80-150 字）
    c. 配图 / 截图 / 例子
    d. 如何用 / 适合谁（30-50 字）

[3] 总结建议（150-250 字）
  → 不要直接结束，给一个"如果只能选一个，我推荐 X"的判断
  → 让读者带走一个明确建议
```

**示范结构**：

> 这是我亲测 38 个 AI 工具后留下的 5 个。
> 
> 不是最热的，是真正每周都在用的。
> 
> ---
> 
> **01｜Claude：替代我 70% 的写作工作**
> 
> [说明 + 截图 + 适合谁]
> 
> **02｜Perplexity：彻底改变了我的搜索习惯**
> 
> [同上]
> 
> ...
> 
> **如果你只能选一个，我推荐 ...**

## 结构 D：拆解爆款型（推荐：行业案例分析 + 审美对象拆解）

**适用**：拆解一部电影 / 一个艺术家 / 一个产品 / 一个爆款现象

**总字数**：2500-4000 字

**结构骨架**：

```
[1] 引出对象 + 现象（150-250 字）
  → 用一个具体画面 / 场景引入要拆解的对象
  → 提出"为什么它这么有效 / 这么美 / 这么火"的问题

[2] 表象观察（300-400 字）
  → 大多数人看到的是什么
  → 用 1-2 个具体细节描述

[3] 深度拆解（1500-2500 字，分 3-4 个维度）
  → 维度 1：[小标题 - 一个本质角度]
  → 维度 2：[另一个角度]
  → 维度 3：[第三个角度]
  → 每个维度配 1-2 张图 + 具体例子

[4] 可复用框架（200-300 字）
  → 把拆解出来的逻辑抽象成 3-4 个可用于其他场景的原则
  → 这是读者能"带走"的部分

[5] 行动建议（100-150 字）
  → 你能怎么用这个框架去看别的作品 / 做别的事
```

**示范开头**：

> 《花样年华》（2000）第 47 分钟，那段 36 秒的慢镜头。
> 
> 张曼玉穿着墨绿色旗袍走出昏黄的走廊，被一束顶光从右上方打下来。
> 
> 这 36 秒已经被无数博主分析过"美"。
> 
> 但我最近重看了王家卫所有作品，发现一件大多数人忽略的事——
> 
> **他不是在"拍美"，他在"做减法"。**
> 
> 而这种减法的逻辑，可以原封不动搬到摄影、穿搭、家居、甚至 PPT 里。

## 结构 E：教程型（推荐：具体技能教学）

**适用**：写 prompt 教程、Midjourney 操作、某个工具的具体用法、某项审美技能的训练

**总字数**：1500-2500 字

**结构骨架**：

```
[1] 痛点共鸣（150-200 字）
  → 你现在面临的问题是什么
  → 这个问题为什么常见 / 难解

[2] 解法承诺（80-150 字）
  → 这篇文章里我会教你一个具体方法
  → 学完你能做到什么

[3] 步骤化教学（每步 200-400 字）
  → Step 1：[标题]
    - 具体怎么做
    - 截图 / 例子
    - 常见错误
  → Step 2：（同上）
  → Step 3：（同上）

[4] 进阶建议（150-250 字）
  → 掌握基本步骤后，下一步怎么做
  → 给一个"再深入"的方向

[5] 结尾召唤（80 字）
  → 直接告诉读者今天就去做哪一件事
```

## 怎么选用哪个结构

| 选题类型 | 推荐结构 |
|---------|---------|
| AIGC 副业 / 落地案例 | A（故事引子） |
| AIGC 工具测评 | C（清单） |
| AIGC 行业观察 | B（观点拆解） |
| AIGC 教程 / Prompt | E（教程） |
| AIGC 学习路径 | C（清单）或 B（观点） |
| 审美 - 大师档案 | D（拆解爆款） |
| 审美 - 训练方法 | B（观点）或 E（教程） |
| 审美 - 跨界观察 | B（观点） |
| 审美 - 个人审美旅程 | A（故事） |
| 审美 - 视觉对象拆解 | D（拆解爆款） |

## 一个万能的检查清单

写完初稿后，按这个清单过一遍结构：

- [ ] 开头 150 字内有钩子吗？（故事 / 反常识 / 数字 / 场景）
- [ ] 每 300-500 字有一个"重新拉回"的设计吗？（小标题 / 加粗 / 转折 / 问句）
- [ ] 每个论点是否都有具体例子撑着？（没例子的论点 = 空话）
- [ ] 是否有一个"可以截图转发"的金句？
- [ ] 结尾有让读者"带走"的东西吗？（行动 / 框架 / 金句）
- [ ] 通篇没有"八股词"（赋能 / 抓手 / 链路 / 颗粒度）吗？
- [ ] 全文有多少处"我"？少于 5 次说明视角太冷
- [ ] 配图够不够？（AIGC 至少 3 张，审美至少 5 张）

任何一项 No → 回去改。
