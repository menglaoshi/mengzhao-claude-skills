# 冷峻代码风 HTML 排版规范

读这个文件的场景：用户要求把公众号文章做成 HTML、优化排版、生成可预览页面、复制到微信公众号编辑器，或希望文章呈现"程序员 / 代码 / 冷峻 / 简约"视觉风格。

---

## 输出目标

做一份**可直接打开、也方便复制到微信公众号编辑器的单文件 HTML**，让文章看起来像一份精修过的技术手记：冷静、克制、有秩序，重点放在正文阅读和观点密度上。

视觉关键词：

- 白色底，正文深色，复制到公众号编辑器后仍然清晰可读
- 等宽字体作为编号、注释、标签和代码块气质
- 正文窄版，移动端优先
- 大量留白，少量细线
- 强调色只用一种冷色，默认青绿色
- 不用 emoji、不用圆角大卡片、不用营销海报式标题

---

## 页面结构

使用 `assets/template.html` 作为骨架，替换以下内容：

1. `<title>`：和正文 H1 保持一致。
2. `.kicker`：文章类别或系列名，例如 `// AIGC NOTE`、`// AESTHETIC LOG`。
3. `h1`：主标题。
4. `.subtitle`：一段 50-90 字导语，说明这篇文章解决什么问题。
5. `.meta`：发布时间、作者、阅读时间等，缺信息就删掉，不要编。
6. `.lead`：开头钩子，承接公众号的前 150 字。
7. `.section`：3-6 个正文段落，每个段落一个清晰小标题。
8. `.quote`：最多 1 个金句块，放全文最值得转发的一句话。
9. `.media-slot`：图片、截图或视频占位说明。没有图就保留 1-3 个明确占位，不要堆空框。
10. `.refs`：可选来源区。没有真实来源就删除整个区块。

## 排版规则

### 标题

- H1 控制在 16-28 个中文字以内。太长就拆成标题 + 副标题。
- 标题不要加书名号外的装饰符号。
- 不使用"震惊"、"重磅"、"必看"等标题党词。

### 正文

- 每段 40-90 个中文字，超过就拆。
- 每 600-900 字插入一个 H2。
- 加粗只用于判断、结论、转折，不超过全文 8 处。
- 数字编号使用 `01`、`02`、`03`，不要用普通有序列表撑结构。
- 需要列表时优先使用短段落，不要连续输出很多 `<li>`。

### 代码风组件

用这些组件建立程序员气质，但不能牺牲可读性：

- `.kicker`：代码注释式栏目标签。
- `.tagline`：一行短标签，例如 `READING MODE: FOCUSED`。
- `.section-index`：等宽编号。
- `.note`：用于 prompt、SOP、检查清单。
- `.code-line`：用于短命令、prompt 片段、公式，不放长代码。

### 图片和媒体

- AIGC 教程/工具测评：建议 3-5 个截图占位。
- 审美拆解：建议 5-8 个图位，占位文字要写清楚"这里放什么图"。
- 图片说明用 `.caption`，写作品名、年份、来源或截图内容。
- 不要为了排版好看放无意义氛围图。

---

## HTML 技术约束

- 单文件 HTML，`UTF-8`。
- 禁止 JavaScript。
- 禁止外链字体、外链 CSS、CDN。
- 不使用 iframe、表单、复杂交互。
- CSS 写在 `<style>` 中；如果用户明确要粘进微信公众号编辑器，尽量保留简单结构和可复制的渲染效果。
- `max-width` 保持 720px 左右；移动端宽度必须不横向滚动。
- 不要依赖 hover、动画或暗色模式媒体查询承载关键信息。

## 交付前检查

- [ ] `<title>`、H1、文件名三者对应。
- [ ] 正文第一屏能看到标题、导语和开头钩子。
- [ ] 所有 `TODO`、占位正文、模板说明都已替换或删除。
- [ ] 没有 markdown 标记残留，如 `##`、`**`、`[链接](url)`。
- [ ] 没有虚构来源、虚构数据、虚构图片。
- [ ] 手机宽度下标题不溢出，表格不撑破页面。
- [ ] 页面可以直接用浏览器打开。
